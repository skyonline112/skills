# Git 커밋 메시지 헬퍼 스킬

이 스킬은 사용자가 Git 커밋 메시지 작성을 요청할 때 활성화되어, 변경 사항을 분석하고 세련된 커밋 메시지를 자동으로 생성합니다.

## 💡 커밋 메시지 표준 형식
```
<type>(<scope>): <subject>

<body>

<footer>
```

## 🛠️ 규칙 및 매핑 가이드
1. **Type (커밋 종류)**:
   - `feat`: 새로운 기능 추가
   - `fix`: 버그 수정
   - `docs`: 문서 수정 (예: README.md, API 문서)
   - `style`: 코드 스타일 변경 (포맷팅, 세미콜론 누락 등, 로직 변경 없음)
   - `refactor`: 코드 리팩토링 (기능 추가나 버그 수정이 없는 구조 개선)
   - `test`: 테스트 코드 추가 및 수정
   - `chore`: 빌드 설정, 패키지 매니저 설정, 프로젝트 설정 변경 등

2. **Subject (제목)**:
   - 50자 이내로 명확하고 간결하게 작성합니다.
   - 명령조(Imperative mood)로 시작하며 마침표(`.`)는 찍지 않습니다.
   - 한국어 또는 영어 중 사용자의 프로젝트 컨벤션에 맞춰 일관되게 제안합니다.

3. **Body (본문) - 선택 사항**:
   - 변경 사항이 복잡한 경우, '무엇을', '왜' 변경했는지 구체적으로 설명합니다.
   - 최대한 간결하고 핵심적인 내용 위주로 개조식 서술을 권장합니다.

## 🚀 에이전트 동작 지침
- 사용자가 `"커밋 메시지 작성해줘"`, `"Git 메시지 추천해줘"`, `"커밋할 거야"` 등의 요청을 하면 이 스킬이 트리거됩니다.
- 에이전트는 먼저 `git diff`나 `git status`를 확인하여 변경사항을 파악합니다.
- 변경된 컨텍스트가 주어지면, 위 규칙에 부합하는 완성도 높은 커밋 메시지 후보군(1~2개)을 즉시 제안합니다.
